## 一、项目定位

OldChat 是一款面向二次元社群的即时通讯 Android 应用，提供私聊、群聊、朋友圈、音乐广场、红包系统、表情包等丰富的社交功能。客户端采用原生 Java 开发，使用大量开源组件（OkHttp、SpongyCastle、Android Support Library）。

| 属性 | 值 |
|------|-----|
| 包名 | `com.im.oldchat` |
| 最低 SDK | Android 2.3 (API 9) |
| 目标 SDK | Android 13 (API 33) |
| 版本号 | 1.2.34 |
| 加密库 | SpongyCastle (Bouncy Castle Android 移植版) |
| 网络库 | OkHttp 3.x |
| 后端协议 | REST + WebSocket (JSON) |

## 四、关键设计决策

### 4.1 端到端加密（E2EE）

- 所有非 `/auth/*` 的 API 请求均使用 ECDH 密钥交换 + AES-GCM 加密
- 客户端与服务端先通过 `/auth/handshake` 交换公钥，协商会话密钥
- 请求体整体加密为 Base64 密文，放在 `X-Enc` 和 `X-Session` 自定义头中

### 4.2 双 Token 认证

- `access_token`：短有效期，用于业务 API 认证
- `refresh_token`：长有效期，用于静默刷新 access_token
- Token 存储在 SharedPreferences (`auth.xml`) 中
- 401 时自动触发刷新流程，连续失败 5 次跳转登录页

### 4.3 实时消息推送

- 使用 WebSocket（ws://）实现消息实时推送
- 支持消息类型：私聊、群聊、撤回、输入状态、在线状态、已读回执
- 指数退避重连策略：300ms → 30s（最大）


### MainActivity

- 应用主 Activity，使用 ViewPager + Fragment 实现底部 Tab 导航
- 四个 Tab：聊天、好友、发现、我的
- 启动时检查 auth token，无效则跳转 LoginActivity
- 定时拉取好友请求和通知

### LoginActivity

- 用户名 + 密码登录
- 支持自定义服务器 Base URL
- 登录成功后存储 token 并跳转 MainActivity

## 六、服务组件

### MessageService
- 前台服务（Foreground Service），保持后台连接
- 管理 WebSocket 连接生命周期
- 在 Android 8+ 上使用 startForegroundService

### MusicPlaybackService
- 前台服务，后台音乐播放
- 通过 Intent Action 控制播放（PLAY/PAUSE/STOP/SEEK）
- 状态通过 Broadcast 广播给 UI
- 带 LRU 缓存机制（缓存上限 220MB / 120 个文件）

### ResourceUploadService
- 前台服务，文件上传
- 支持进度通知
- 上传至 `/resources/upload` 接口

## 七、项目中的代码混淆特征

源代码经过 ProGuard/R8 混淆，特征包括：
- 类名和变量名被重命名为单字母（a, b, c...）
- 资源 ID 映射到 `f0.f`, `f0.e`, `f0.d`, `f0.g` 等常量类
- 实际逻辑集中在 `o0`, `g0`, `k`, `l0` 等混淆包中
- `AbstractCxxxx` 表示抽象类，`Cxxxx` 表示普通类

## 八、已知限制与技术债务

- 仍使用已废弃的 `AsyncTask` 做异步操作
- 使用 `android.support.*` 而非 AndroidX
- `minSdk=9` 保留了大量兼容代码
- 部分 I/O 操作未在主线程做严格隔离
- 音乐缓存 LRU 实现在 `LinkedHashMap` 中直接硬编码，扩展性有限
