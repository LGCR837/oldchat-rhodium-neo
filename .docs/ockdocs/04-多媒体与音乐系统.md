# OldChat Android 客户端 — 多媒体与音乐系统

> 本章详细分析客户端的音乐播放、资源上传和多媒体处理机制。

```
├─────────────────────────────────────────────────┤

│  网络接口：获取歌曲信息、点赞、评论                  │
└─────────────────────────────────────────────────┘
```

### 2.4 音频焦点管理

```java
@Override
public void onAudioFocusChange(int focusChange) {
    if (focusChange == AudioManager.AUDIOFOCUS_LOSS_TRANSIENT) {
        // 临时失去焦点 → 暂停，记录状态
        isTemporarilyLost = isPlaying();
        pause();
    } else if (focusChange == AudioManager.AUDIOFOCUS_LOSS) {
        // 永久失去焦点 → 暂停
        isTemporarilyLost = false;
        pause();
    } else if (focusChange == AudioManager.AUDIOFOCUS_GAIN) {
        // 重新获得焦点 → 恢复播放
        if (isTemporarilyLost) {
            resume();
            isTemporarilyLost = false;
        }
    }
}
```

## 三、音乐缓存策略

### 3.1 缓存架构

```java
// LRU 缓存，最多缓存 18 个歌曲元信息
public static final LinkedHashMap songInfoCache = new LinkedHashMap(24, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry entry) {
        return size() > 18;
    }
};
```

### 3.2 文件缓存管理

- 缓存位置：`getCacheDir()/music/`
- 缓存上限：220 MB 或 120 个文件
- 清理策略：LRU（最近最少使用），后台线程定期扫描
- 缓存文件名：基于 URL 的 SHA-256 哈希

```java
// 缓存清理逻辑
File[] files = cacheDir.listFiles();
long totalSize = 0;
int fileCount = 0;
for (File file : files) {
    totalSize += file.length();
    fileCount++;
}
// 如果超过 220MB 或 120 个文件，删除最旧的文件
```

### 3.3 播放容错机制

```
尝试播放 URL
    │
    ├─ 在线播放 → 失败 → 检查本地缓存
    │                           │
    │                           ├─ 有 → 播放本地文件
    │                           └─ 无 → 尝试备用 URL
    │                                      │
    │                                      └─ 也失败 → 显示 "播放失败"
    │
    └─ 成功 → 异步缓存到本地（后台线程）
```

## 四、音乐广场

### MusicPlazaActivity

音乐广场是一个独立的 Activity，提供：
- 歌曲列表（分页加载，limit/offset）
- 搜索功能（`/music/search`）
- 歌曲上传（`POST /music/plaza`）
- 点赞/取消点赞
- 评论查看
- 点击歌曲 → 打开 MusicPlayerActivity 或直接播放

API 接口：

| 端点 | 方法 | 功能 |
|------|------|------|
| `/music/plaza` | GET | 获取音乐广场列表 |
| `/music/plaza` | POST | 上传歌曲到广场 |
| `/music/search` | GET | 搜索歌曲 |
| `/music/like` | POST | 点赞歌曲 |
| `/music/unlike` | POST | 取消点赞 |
| `/music/comments` | GET | 获取评论 |
| `/music/comment` | POST | 发表评论 |

## 五、ResourceUploadService 资源上传

### 5.1 上传流程

```
用户选择文件（图片/视频/文件）
    │
    ├─ 1. 检查权限（Android 13 READ_MEDIA_*）
    │
    ├─ 2. 启动 ResourceUploadService
    │     • Intent extra: section_id, content_uri, mime_type, total_bytes
    │
    ├─ 3. 前台服务通知（进度条 + 文件信息）
    │
    ├─ 4. Multipart POST /resources/upload
    │     • 流式上传（从 ContentResolver 读取）
    │     • 进度回调 → 更新通知栏
    │
    ├─ 5. 成功 → 返回 media_url → 发出消息
    │
    └─ 6. 失败 → 通知栏显示失败
             → 停止服务
```

### 5.2 上传 API

```http
POST /resources/upload
Authorization: Bearer {token}
Content-Type: multipart/form-data

--boundary
Content-Disposition: form-data; name="file"; filename="xxx.jpg"
Content-Type: image/jpeg

[binary data]
--boundary
Content-Disposition: form-data; name="section_id"

{section_id}
--boundary--
```
