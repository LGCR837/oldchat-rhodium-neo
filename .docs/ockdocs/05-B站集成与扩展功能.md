# OldChat Android 客户端 — B站集成与扩展功能

> 本章分析客户端的 Bilibili API 集成、"Old View" 旧版视图系统，以及红包、公堂、表情包等特色功能。

---

## 一、B站集成 (com.im.oldchat.bili)

OldChat 客户端深度集成了 Bilibili 开放平台 API，允许用户在应用内浏览 B站 视频、管理收藏夹和查看评论。

### 1.1 BiliModels 数据模型

包 `com.im.oldchat.bili` 包含完整的 B站 API 响应模型：

| 模型类 | 对应 B站 API |
|--------|-------------|
| `VideoDetailResult` | 视频详情 (`/x/web-interface/view`) |
| `VideoDetailData` | 视频元信息（标题、描述、UP主、分P） |
| `VideoOwner` | UP 主信息（mid, name, face） |
| `VideoStat` | 播放统计数据（播放、弹幕、点赞、评论数） |
| `SearchResult` | 搜索结果 |
| `SearchData` | 搜索结果列表（视频、用户、番剧） |
| `CommentResult` | 评论列表 |
| `CommentReply` | 单条评论（含回复嵌套） |
| `CommentMember` | 评论用户信息 |
| `CommentPage` | 评论分页 |
| `FavFolderResult` | 收藏夹列表 |
| `FavFolder` | 单个收藏夹（标题、媒体数、状态） |
| `FavMedia` | 收藏的视频条目（BV号、封面、时长） |
| `BiliResponse<T>` | 通用 B站 API 响应包装器 |
| `TokenInfo` | OAuth Token 信息 |
| `BiliCookie` | Cookie 信息 |
| `WbiImg` | WBI 签名用图片 |

### 1.2 B站 API 功能

通过解耦的 a-f 辅助类实现：

| 功能类 | 功能 |
|--------|------|
| `a.java` | B站 API 基础调用 |
| `b.java` | 视频详情获取 |
| `c.java` | 搜索 |
| `d.java` | 收藏夹管理 |
| `e.java` | 评论获取 |
| `f.java` | 用户信息 |

### 1.3 应用场景

OldChat 中将 B站集成作为 "Old View"（旧版视图）功能呈现，包括：

- **VideoDetailActivity** — 播放 B站 视频（调用系统播放器或 WebView）
- **OldViewFavoritesActivity** — 查看 B站 收藏夹
- **OldViewHistoryActivity** — 查看观看历史
- **OldViewUpProfileActivity** — 查看 UP 主主页

## 二、Old View 旧版视图系统

Old View 是一套独立的 B站 内容浏览子系统，包含完整的视频浏览、搜索和收藏管理体验。

### 2.1 组件结构

```
OldViewActivity (主列表，多种内容类型)
├── OldViewVideoDetailActivity (视频详情页)
│   ├── 视频播放
│   ├── 评论列表
│   └── 相关推荐
├── OldViewVideoFullActivity (全屏播放)
├── OldViewUpProfileActivity (UP 主主页)
├── OldViewFavoritesActivity (收藏夹列表)
│   └── OldViewFavoriteDetailActivity (收藏夹内容)
└── OldViewHistoryActivity (观看历史)
```

### 2.2 Old View 布局

布局文件包括：

| 文件 | 用途 |
|------|------|
| `activity_old_view.xml` | 主列表页 |
| `activity_old_view_detail.xml` | 视频详情 |
| `activity_old_view_fullscreen.xml` | 全屏播放 |
| `activity_old_view_list.xml` | 列表页 |
| `activity_old_view_up_profile.xml` | UP 主页 |

## 三、红包系统

OldChat 实现了完整的社交红包功能。

### 3.1 红包类型

| 类型 | 说明 |
|------|------|
| 普通红包 | 固定金额，每人一份 |
| 拼手气红包 | 随机金额分配 |

### 3.2 API 接口

| 端点 | 方法 | 功能 |
|------|------|------|
| `/red/packet/send` | POST | 发送红包 |
| `/red/packet/open` | POST | 拆开红包 |
| `/red/packet/detail` | GET | 红包详情（领取记录） |

### 3.3 红包消息体

红包在聊天中表现为特殊类型的消息 (`msg_type: "red_packet"`)：

```json
{
    "msg_type": "red_packet",
    "body": {
        "packet_id": "uuid",
        "title": "恭喜发财",
        "total_amount": 100,
        "total_count": 5,
        "cover_url": "/red_packet/default_cover.png"
    }
}
```

### 3.4 UI 组件

红包相关 Activity：

| Activity | 功能 |
|----------|------|
| `RedPacketSendActivity` | 发送红包（设置金额、个数、祝福语） |
| `RedPacketOpenActivity` | 拆红包动画 + 结果展示 |
| `RedPacketDetailActivity` | 红包详情（谁领了多少） |

红包 UI 拥有独特的视觉风格：
- 红包封面动画（发光、倾斜）
- 开红包的"拆"按钮动画
- 领取记录列表

## 四、公堂/审判系统

OldChat 包含一个有趣的 "公堂"（Public Court）功能，可能是一个社区纠纷仲裁系统。

### 4.1 API 接口

| 端点 | 方法 | 功能 |
|------|------|------|
| `/court/cases` | GET | 获取案件列表 |
| `/court/case/{id}` | GET | 案件详情 |
| `/court/vote` | POST | 投票 |
| `/court/statement` | POST | 提交陈述 |
| `/court/judgement` | GET | 判决结果 |

### 4.2 布局

| 布局 | 用途 |
|------|------|
| `activity_public_court.xml` | 公堂主页（案件列表） |
| `activity_public_court_case_detail.xml` | 案件详情 |
| `dialog_public_court_statement.xml` | 提交陈述对话框 |

## 六、每日签到系统

### 6.1 API 接口

| 端点 | 方法 | 功能 |
|------|------|------|
| `/me/checkin/wall` | GET | 签到墙（签到状态 + 精选留言） |
| `/me/checkin` | POST | 执行签到 |
| `/me/checkin/wall` | POST | 发布签到留言 |
| `/me/checkin/wall/comments` | GET | 签到墙评论 |

### 6.2 签到墙数据模型

```json
{
    "checkin_count": 7,
    "checked_in": true,
    "already_posted": false,
    "my_post": {},
    "featured_messages": [
        {
            "uid": "user_uid",
            "username": "用户名",
            "avatar_url": "...",
            "content_text": "今天也是美好的一天！",
            "image_url": "...",
            "likes": 5,
            "created_at": 1234567890
        }
    ]
}
```

### 6.3 Activity

- `DailyCheckInActivity` — 签到主界面（签到墙 + 月历）
- `DailyCheckInCommentsActivity` — 签到留言评论
